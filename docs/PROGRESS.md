## 2026-07-10 - Macro C C-PR4 MERGED (per-node retry policy + dead-letter)

- C-PR4 (`task/v2c-04-retry-timeout` -> `task/v2c-graph-executor`, PR #59) MERGED (merge commit `60a466c`). Task 10 of the Macro C plan.
- **Retry attribute + policy**: new `src/Executor/Attributes/Retry` (`@api`, `TARGET_METHOD|TARGET_CLASS`, `tries`/`backoff`/`timeout`) + pure `src/Executor/RetryPolicy` (`@api`) value object. `tries` clamps to a minimum of 1 (diverges deliberately from Laravel's job-level "0 = unlimited" so a poison node can never loop forever). `backoff` is an int (fixed) or a `list<int>` (per-attempt, clamped to the last entry once attempts exceed the schedule). `timeout` is a documented POST-HOC wall-clock check only (a `NodeTimeoutException` recorded after the attempt returns) — true preemptive timeout is deferred to the queue worker in C-PR5. `RetryPolicy::fromAttribute()` + `withConfig()` let a graph node's `config['retry']` override the attribute; `toArray()` advertises the resolved policy to the node catalog.
- **Executor integration**: `NodeExecutor::execute()` gains a retry loop around a new `runAttempt()` seam (reuses the `Failed->Running` legal transition from C-PR1). Dead-letter (`NodeState::DeadLetter`) is reached only when a REAL retry budget (`tries > 1`) is exhausted; a single-attempt failure (`tries == 1`, the default for a node with no `#[Retry]`) stays `Failed`. Backoff between retries uses `backoffForAttempt($attempts)` (the Nth retry consumes the Nth schedule entry — 1-based); dry runs skip the real `Sleep` so a dry-run of a failing node has zero executor-driven delay. `NodeDefinitionFactory::retryFor()` reads `#[Retry]` from `execute()` (preferred) or the class, wrapping any malformed attribute as `InvalidNodeDefinitionException`. `NodeDefinition` carries an optional `?RetryPolicy $retry`, serialized by `toArray()` only when present.
